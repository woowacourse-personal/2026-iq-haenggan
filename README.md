# 썰풀이 (ssulpuri)

> 요약은 정보를 줄이고, 썰풀이는 정보를 기억에 남게 한다.

딱딱한 기사·정보성 글을 넣으면, 인물·갈등·전개가 있는 **이야기**로 재구성해주는 서비스.
핵심 제약: **사실은 불변 계약** — 재미를 위해 지어내지 않는다.

## 실행 방법

```bash
# 1. 가상환경 생성 및 의존성 설치
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. API 키 설정
cp .env.example .env
# .env 파일을 열어 ANTHROPIC_API_KEY 입력

# 3. 서버 실행
uvicorn app.main:app --reload

# 4. 브라우저에서 열기
# http://localhost:8000
```

## 아키텍처 — 4단계 서사화 파이프라인

```
[기사] → ① 사실 추출 → ② 서사 설계 → ③ 각색 → ④ 사실 대조 → [이야기]
          (Haiku)        (Sonnet)      (Sonnet)   (Haiku)      ↑
          사실 목록 = 불변 계약 ─────────────────────────────────┘
                                        불일치 발견 시 수정 재생성 1회
```

| 단계 | 파일 | 역할 | 모델 |
|---|---|---|---|
| ① 사실 추출 | `app/pipeline/extract.py` | 기사에서 사건·수치·발언을 구조화. 이후 모든 단계의 불변 계약 | 저비용 |
| ② 서사 설계 | `app/pipeline/arc.py` | 사실을 이야기 뼈대(배경→발단→전개→절정→의미)에 배치 | 고품질 |
| ③ 각색 | `app/pipeline/narrate.py` | 제약 걸린 생성 — 사실 창작 금지, 추정 명시, 투자권유 금지 | 고품질 |
| ④ 사실 대조 | `app/pipeline/verify.py` | 이야기 속 주장 ↔ 사실 목록 대조, 불일치 시 수정 트리거 | 저비용 |

모든 중간 산출물(사실 목록, 설계도, 검증 리포트)이 UI에 노출된다.
→ 단계별 품질을 눈으로 확인하며 프롬프트를 업그레이드하는 실험 구조.

## 업그레이드 로드맵

- **v0.1 (현재)**: 텍스트/URL 입력 → 4단계 파이프라인 → 이야기 + 검증 리포트
- **v0.2 후보**
  - SSE로 단계별 실시간 진행 표시 (현재는 근사 타이머)
  - 이야기 유형 추가 (대화체, 타임라인형)
  - 파이프라인 구성 A/B 비교 모드 (프롬프트 실험용)
- **v0.3 후보**
  - 회상 테스트 실험 모드 (원문 vs 이야기, 24시간 뒤 퀴즈)
  - TTS 음성 버전
- 상세 기획: [docs/planning.md](docs/planning.md)

## 검증 우선순위 (기획서 §3)

1. **H1 가치**: 이야기로 읽으면 24시간 뒤 회상률이 더 높은가 → `experiments/` 참고
2. **H2 품질**: LLM 각색이 "원문보다 낫다"고 느껴지는가
3. **H3 신뢰**: 사실 왜곡이 ④단계에서 잡히는가
4. **H4 행동**: 시키지 않아도 반복 사용하는가

## 원칙

- 원문 기사는 저장하지 않는다 (파이프라인 입력으로만 사용, 저작권 §7)
- 추정은 반드시 추정으로 표기한다
- 종목 매수/매도 권유 표현은 생성하지 않는다 — 이해를 돕는 콘텐츠이지 투자 조언이 아니다
